## Codebase Patterns
- WebSocket gateway at `apps/tictactoe-api/src/modules/game/game.gateway.ts` — uses @nestjs/websockets v10 + @nestjs/platform-socket.io v10 (must match NestJS core version)
- Redis adapter configured in `afterInit()` via OnGatewayInit lifecycle hook — uses ioredis pub/sub clients with `pubClient.duplicate()` for sub client
- GameGateway is a provider in GameModule, injected into GameService for broadcasting PvP game updates
- Use SnakeNamingStrategy: camelCase in TypeScript -> snake_case in DB columns (automatic via TypeORM config)
- Migrations are in `apps/tictactoe-api/src/migrations/` with timestamp prefix
- Run migrations with `pnpm typeorm:migration:run` from `apps/tictactoe-api/`
- PostgreSQL on port 54322 (not default 5432) to avoid conflicts
- Game entity at `apps/tictactoe-api/src/modules/game/game.entity.ts`
- Types (CellValue, GameMove, GameStatus, GameMode) are defined in the entity file AND in `apps/tictactoe-api/src/lib/game/types.ts`
- Pure game logic functions in `apps/tictactoe-api/src/lib/game/gameLogic.ts` — import types from `./types`
- Typecheck: `pnpm typecheck` from `apps/tictactoe-api/`
- Pre-commit hooks run eslint and prettier via lint-staged
- DTOs in `apps/tictactoe-api/src/modules/game/dto/` — use class-validator for validation
- GameService has `toGameStateDto()` private method to map entity -> DTO (does NOT expose player tokens)
- GameController uses `@Controller('api/games')` prefix
- Use `@UsePipes(new ValidationPipe({ whitelist: true }))` per-endpoint for input validation
- For transactional operations with row locking: inject `DataSource`, create queryRunner, use `setLock('pessimistic_write')` for SELECT FOR UPDATE
- Board is JSONB in DB — cast to `Board` type from `../../lib/game/types` when passing to game logic functions
- AI module at `apps/tictactoe-api/src/modules/ai/` — AiService.generateMove(board) invokes LangGraph pipeline
- AI files use kebab-case filenames: move-graph.ts, graph-state.ts, move-validator.ts, move-prompt.ts, llm.ts, nodes.ts
- NestJS upgraded to v10, TypeScript upgraded to v5.9 — required for compatibility with zod v4 (transitive dep from @langchain/core)
- AWS Bedrock env vars: AI_AWS_BEDROCK_ACCESS_KEY_ID, AI_AWS_BEDROCK_SECRET_ACCESS_KEY, AI_AWS_BEDROCK_REGION
- Two DB connections: TypeORM uses postgres superuser (full access), PostGraphile uses postgraphile_user (SELECT only) — configured in separate config files (typeorm.ts vs graphile.ts)
- PostGraphile credentials: POSTGRAPHILE_USER and POSTGRAPHILE_PASSWORD env vars (defaults: postgraphile_user / postgraphile)

  - `GRANT CONNECT ON DATABASE` and `GRANT USAGE ON SCHEMA public` are needed in addition to table-level GRANT SELECT — without these the role can't even access the schema
  - Verified with psql: SELECT works, INSERT/UPDATE/DELETE all return "permission denied for table game"
  - The PostGraphile config in graphile.ts overrides only `user` and `password` fields — host, port, and database are shared from the common postgres config
